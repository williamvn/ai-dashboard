# Product UI Vision

## Product Purpose

Help engineering leaders understand AI usage, spend, and quality across their teams.

This should feel like a modern analytics SaaS product.

Priorities:

- clarity over decoration

- useful metrics over vanity metrics

- professional tone

- fast navigation

- data dense but readable

---

## Main Navigation

1. Landing Page
2. Dashboard
3. Simulate
4. About

---

## 1. Landing Page

Purpose: select the organisation to analyse.

This is not a marketing homepage.

Key elements:

- organisation list/cards
- recent organisations
- search if needed
- select organisation CTA

When an organisation is selected, navigate to Dashboard.

---

## 2. Dashboard

After selecting an organisation, user lands here.

Dashboard has 3 main sections/tabs:

### Usage

Shows how AI is being used.

Examples:

- total calls
- active engineers
- calls over time
- most active teams
- adoption trends

### Cost

Shows consumption/cost efficiency.

Examples:

- total spend
- spend over time
- cost by team
- tokens by agent
- cost by engineer

### Impact

Shows business value.

Examples:

- estimated time saved
- throughput improvement
- productivity trends
- ROI indicators

---

## 3. Simulate

Allows modelling usage behaviour over a selected period.

Concept:

Each engineer has a probability of making calls.
Some engineers are more active than others.

Inputs:

- organisation
- time range
- team size
- activity profile
- usage probabilities

Outputs:

- projected usage
- projected cost
- estimated impact
- charts and comparisons

---

## UX Rules

- Keep navigation obvious
- Avoid unnecessary modals
- Prefer charts with meaning
- Avoid empty decorative cards
- Tables should be readable
- Every chart should answer a business question
