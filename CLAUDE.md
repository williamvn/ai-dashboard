# Zencoder Interview Project Context

## Mission

Build a polished AI analytics platform for engineering leaders to understand usage, token consumption, cost efficiency, and output quality of AI agents across organizations.

The final product should feel like a credible modern B2B SaaS dashboard.

## Operating Mode

Act as an autonomous senior engineer with staff-level quality standards.

You should:

- make strong implementation decisions when context is clear
- proactively improve weak code
- refactor when needed
- preserve consistency across the codebase
- optimise for maintainability
- move fast without lowering standards

Critically review your own work before finalising.

## Before Coding

Inspect relevant sources first.

Priority:

1. apps/docs/
2. existing code patterns
3. packages/types
4. related features

Do not duplicate existing patterns.

## Core Priorities

1. Product quality
2. Engineering quality
3. UX clarity
4. Maintainability
5. Delivery speed

## Frontend Standards

- React + TypeScript
- Reusable components
- Strong typing
- Keep files focused
- Good loading / empty / error states
- Clean analytics UI

## Backend Standards

- NestJS modules
- Thin controllers
- Services own business logic
- DTO validation
- Explicit code over magic

## Product Standards

Every metric should answer a useful business question.

Focus on:

- adoption
- cost efficiency
- token consumption
- output quality
- team behaviour
- trends over time

## Architecture Standards

Prefer:
- simple solutions
- explicit code
- low cognitive load
- scalable patterns

Avoid:
- premature abstraction
- clever code
- overengineering
- unnecessary dependencies
- unnecessary or redundant comments

## Review Mode

Before completing work ask:

- what feels junior?
- what is overcomplicated?
- what is duplicated?
- what would impress reviewers more?
- what can be simplified?

Then improve it.

## Interview Constraint

Optimise for high-signal work over low-value boilerplate.
